# Copilot Anatomy

**Visibility:** This repository is public.

## Purpose

A structured reference to the services and features making up the
Microsoft 365 Copilot suite — one article per service/feature, covering
what it is, what it depends on, its known limitations, and how it relates
to other features.

## Sourcing

Content here draws **only on externally-available source material** —
public Microsoft/vendor documentation, announcements, and similar publicly
published sources. Every article cites its source(s) in a `sources` list
in frontmatter; a feature covered by more than one source has its
information merged into a single account rather than kept as separate
per-source sections. Where sources disagree on a specific point, that
point is flagged with a bare `[?]` marker rather than the article naming
which source says what.

Any added human commentary is kept in a clearly separated **Insight**
section at the end of each article, visually distinguished (blockquoted)
from the source-derived content above it — so it's always obvious which
parts are sourced fact and which are opinion/analysis layered on top.

See `CLAUDE.md` for the full schema and conventions used to maintain this
repo.

## Disclaimer

This project is primarily an experiment in combining an LLM with a
structured wiki — testing how well an AI agent can ingest, synthesize, and
maintain a reference knowledge base over time. It is not a professional
publication, and no article here should be taken as commentary on the
reliability, accuracy, or completeness of the sources it draws on.

Nor should anything in this repository be relied on as an accurate or
complete account of Microsoft 365 Copilot, Agent 365, or any other
product/service it references. Content is generated by an LLM from
third-party source material and may be incomplete, outdated, or wrong.
For anything that matters, verify against Microsoft's own official
documentation.

Microsoft.
